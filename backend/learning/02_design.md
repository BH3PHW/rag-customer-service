## 2.1 什么是系统设计？

### 📖 定义

**系统设计**是将需求转化为技术实现方案的过程。

简单说，就是回答：
- **架构怎么搭？** - 整体结构设计
- **模块怎么分？** - 功能模块划分
- **数据怎么流？** - 数据流设计

### 🎨 设计原则

```
好的系统设计应该遵循：
┌─────────────────────────────────┐
│  1. 高内聚低耦合                │
│  2. 单一职责原则                │
│  3. 可扩展性                    │
│  4. 可维护性                    │
└─────────────────────────────────┘
```

## 2.2 抽象化思维

### 🧠 什么是抽象化？

**抽象化**就是从具体问题中提取核心特征，忽略次要细节的过程。

举例：
- 当设计聊天功能时，我们不需要一开始就考虑数据库细节
- 先抽象出"发送消息"、"获取回复"这样的核心接口

### 🔍 抽象化步骤

```
具体问题 → 提取核心特征 → 定义抽象接口 → 实现细节
```

### 📝 抽象化示例

**需求：** 用户发送消息，AI回复

**抽象化：**
```python
# 抽象接口
class ChatService:
    def send_message(self, user_id: str, message: str) -> str:
        """发送消息并返回回复"""
        pass
    
    def get_session(self, session_id: str) -> dict:
        """获取会话信息"""
        pass
```

**实现细节：**
```python
# 具体实现
class RAGChatService(ChatService):
    def send_message(self, user_id: str, message: str) -> str:
        # 1. 检索知识库
        context = self.retrieve_context(message)
        # 2. 调用LLM
        response = self.generate_response(message, context)
        return response
```

## 2.3 架构设计

### 🏗️ 架构选择

**为什么选择微服务架构？**

| 对比项 | 单体架构 | 微服务架构 |
|--------|----------|------------|
| 复杂度 | 低 | 高 |
| 扩展性 | 差 | 好 |
| 部署 | 简单 | 复杂 |
| 容错性 | 差 | 好 |
| 团队协作 | 难 | 易 |

```

    │
    ├──► 调用LLM生成回答
    │
    └──► 返回响应
```

## 2.4 模块划分

### 📦 后端服务模块

| 服务 | 职责 | 核心功能 |
|------|------|----------|
| **api-gateway** | 统一入口 | 路由转发、安全控制 |
| **user-service** | 用户管理 | 认证、授权、企业管理 |
| **chat-service** | 聊天处理 | RAG检索、LLM调用、流式输出 |
| **knowledge-service** | 知识库 | 文档管理、向量存储、语义检索 |
| **alert-service** | 告警服务 | 敏感内容检测、通知 |
| **channel-service** | 渠道接入 | 多渠道消息路由 |

## 2.5 数据库设计

### 🗃️ 数据库选择

| 数据库 | 用途 | 原因 |
|--------|------|------|
| PostgreSQL | 主数据存储 | 支持复杂查询、事务 |
| Redis | 缓存/会话 | 高性能、支持过期 |
| ChromaDB | 向量存储 | 支持语义检索 |

### 📊 核心数据表

**用户表 (users):**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 用户ID |
| email | VARCHAR | 邮箱 |
| name | VARCHAR | 姓名 |
| password | VARCHAR | 加密密码 |
| role | ENUM | 用户角色 |
| enterprise_id | UUID | 所属企业 |

**会话表 (chat_sessions):**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 会话ID |
| user_id | UUID | 用户ID |
| enterprise_id | UUID | 企业ID |
| status | ENUM | 会话状态 |
| created_at | TIMESTAMP | 创建时间 |
